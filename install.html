INSTALLATION

Préparation de la carte SD

La préparation de la carte SD s'effectue avec l'outil Pi Imager.

A partir d'un PC Windows, télécharger Raspberry Pi Imager et l'installer.

A partir d'un Raspberry, Pi Imager est déjà installé.

Liste des actions
- Lancez Pi Imager
- Allez dans Choisir l'OS et choisissez l'OS selon le Raspberry destiné à accueillir MyChronoGPS (32-bit tout type de rpi, 64-bit rpi 3 , 4, 400, 0W V2) de préférence avec l'environnement de bureau.
- Allez dans les paramètres de Pi Imager et effectuez les actions
- Cochez la case Activer SSH
(La case Définir nom d'utilisateur et mot de passe est automatiquement cochée)
- Laissez Nom d'utilisateur à pi et entrez le Mot de passe, chrono par exemple, notez le password.
- Cochez la case Configurer le Wi-Fi et entrez les paramètres de votre réseau Wifi local.
- Cochez la case Définir les réglages locaux et entrez votre zone géographique et le type de clavier.
- Sauvegardez les paramètres.
- Installez la carte SD dans le lecteur et allez dans Choisir le Stockage.
Le lecteur est automatiquement sélectionné, s'il y en a plusieurs choisir le lecteur dans lequel la carte SD cible est installée.
- Appuyez sur Ecrire et patientez jusqu'à la fin de l'écriture de la carte SD.
- Retirez la carte SD et installez-là dans le Raspberry cible.

Installation de la carte SD

La suite de l'installation s'effectue à partir du Raspberry ou via une connexion SSH.
- Installez la carte SD dans le lecteur du Raspberry cible.
- Démarrez le Raspberry.

Aide à l'installation via SSH (dans le cas du Rpi zero W par exemple)

Ajoutez un fichier vide nommé SSH (sans extension) dans la partition Boot.

Ajoutez un fichier wpa_supplicant.conf dans la partition Boot avec le contenu suivant (un exemple existe dans le github du projet) :

nano wpa_supplicant.conf
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1
network={
ssid="first access point"
psk="acces key"
key_mgmt=WPA-PSK
}

Remplacez first access point par le ssid du réseau wifi de votre box.

Remplacez acces key par le mot de passe wifi de votre box.

Ajoutez autant de groupe network que vous avez de réseau wifi (box, smartphone en point d'accès, etc).

Sauvegardez le fichier.

Le Raspberry sera configuré automatiquement pour démarrer avec le réseau wifi de votre box.

Il suffira ensuite de répérer l'adresse ip du Raspberry pour s'y connecter en SSH

Verification et/ou Configuration du Raspberry

Passer la commande :
sudo raspi-config
Allez dans
3 - Interface Options
P4 SPI => Oui
P5 I2C => Oui
P6 Serial Port => shell:non / serial port hardware:oui
A la fin de la configuration faire un reboot
sudo reboot
Installer les outils I2C pour Python (déjà installé sur les versions récentes)
Mettre à jour le système (s'il y a longtemps que ça n'a pas été fait)
sudo apt update
sudo apt upgrade -y
L’installation terminée, on peut déjà tester si les modules sont bien chargés
Vérifier l’adresse du périphérique avec i2cdetect
sudo i2cdetect -y 1 # pour les anciens modèles : sudo i2cdetect -y 0
Si besoin, installer l'interface série
sudo pip install pyserial
sudo pip install gpio

Ensuite, il est nécessaire d'installer un serveur web pour avoir accès aux outils de visualisation des sessions et de la gestion des circuits. Il est également nécessaire pour l'affichage du tableau de bord sur un écran LCD HDMI (minimum 3.5" 480x360 recommandé).

Installation de MyChronoGPS sur le Raspberry Pi

A partir du dossier racine utilisateur (/home/pi par exemple) lancez les commandes suivantes :
sudo rm -rf MyChronoGPS
git clone https://github.com/jfk93-fr/MyChronoGPS.git
chmod -R 755 MyChronoGPS
cd MyChronoGPS/
La procédure d'installation proprement dite de MyChronoGPS s'effectue par la commande :
sh MyChronoGPS.sh

Plusieurs questions sont posées.

Si Lighttpd n'est pas installé, la procédure d'installation demande si on doit l'installer, c'est préférable, aussi répondre Y

La procédure d'installation demande si on a une clé API pour GoogleMaps et si on veut l'utiliser.
Si oui, le fichier key.js est édité et il faut remplacer 'Your_API_Key' par la valeur de votre clé.

A la fin de l'installation, redémarrer le système
sudo reboot

Installation physique du GPS

Le fichier paramètre de MyChronoGPS est configuré pour un GPS relié au Raspberry via une liaison série.
Si c'est le cas, passez la commande :
cat /dev/serial0

les trames envoyées par le GPS doivent s'afficher sur la console.

Si le GPS est relié via le port USB, rechercher le fichier associé.
Le fichier devrait avoir un nom sous la forme /etc/ttyACM*

Pour afficher le nom, passez la commande
ls /dev/ttyA*

exemple :

pi@raspberrypi:~ $ ls /dev/ttyA*
/dev/ttyACM0 /dev/ttyAMA0
pi@raspberrypi:~ $

Il faut essayer de trouver le bon fichier en passant la commande :

pi@raspberrypi:~ $ cat /dev/ttyAMA0
pi@raspberrypi:~ $

Il n'y a rien en retour, essayez avec :

pi@raspberrypi:~ $ cat /dev/ttyACM0
$GNTXT,01,01,02,u-blox AG - www.u-blox.com*4E
$GNTXT,01,01,02,HW UBX-M8030 00080000*60
$GNTXT,01,01,02,ROM CORE 3.01 (107888)*2B

Ce sont les trames NMEA envoyées par le GPS /dev/ttyACM0

Mettre à jour le fichier paramètre de MyChronoGPS en passant la commande

sudo nano /home/pi/MyChronoGPS/parms/params.json

Rechercher les lignes contenant "GPSPort"

"#GPSPort": "port rattaché au GPS (defaut; /dev/serial0)",
"GPSPort": "/dev/serial0",
"GPSPort": "/dev/ttyACM1",
"GPSPort": "/dev/ttyACM0",

C'est la dernière ligne qui est prise en compte.
Dans l'exemple ci-dessus, le GPS est associé au fichier /dev/ttyACM0

Installation écran TFT LCD Touch Screen

Configurez l’écran 3.5 inch TFT LCD Touch Screen avec le Raspberry Pi en passant les commandes :
sudo rm -rf LCD-show
git clone https://github.com/goodtft/LCD-show.git
chmod -R 755 LCD-show
cd LCD-show/
sudo ./LCD35-show
Après redémarrage du Raspberry Pi vous devriez voir apparaitre le desktop de Raspbian sur votre petit écran.

Pour faire pivoter l'affichage de 180° :
cd LCD-show/
sudo ./rotate.sh 180
Après redémarrage du Raspberry Pi, l'affichage de l'écran a pivoté de 180°.

Installation afficheurs Nextion

Dans le dossier Nextion, prendre le fichier MyChronoGPS_Nextion_Display.HMI générique ou correspondant au type d'écran utilisé et l'ouvrir dans Nextion Editor

A partir de Nextion Editor, transférer (upload) le fichier MyChronoGPS_Nextion_Display.HMI vers l'écran Nextion connecté.

Mettre à jour le fichier paramètre de MyChronoGPS en passant la commande

sudo nano /home/pi/MyChronoGPS/parms/params.json

Rechercher les lignes contenant "ScreenCmd"

"#ScreenCmd": "program name associated with display 1",
"#ScreenCmd_": "pgm dispos: MyChronoGPS_OLED MyChronoGPS_LCD MyChronoGPS_TE2in13V3 MyChronoGPS_Nextion_Display MyChronoGPS_Nextion_Dashboard",
"ScreenCmd": "MyChronoGPS_Dummy",

C'est la dernière ligne qui est prise en compte.
Remplacer "MyChronoGPS_Dummy" par "MyChronoGPS_Nextion_Dashboard" ou par "MyChronoGPS_Nextion_Display"

Le programme "MyChronoGPS_Nextion_Display affiche des données similaires aux autres programmes d'afficheurs.

Le programme "MyChronoGPS_Nextion_Dashboard affiche un tableau de bord.

Le fichier .HMI peut-être personnalisé, juste repérer les variables qui sont envoyées vers l'écran

Lancement de MyChronoGPS au démarrage du Raspberry Pi

Editer le fichier /etc/rc.local :
sudo nano /etc/rc.local
Avant la ligne exit 0 insérer la commande pour démarrer MyChronoGPS au démarrage du Raspberry Pi :
sh /home/pi/MyChronoGPS/start_gps.sh
Si nom utilisateur # pi => remplacer /pi par /nom utilisateur

Lancement du tableau de bord au démarrage du Raspberry Pi

Pour lancer le tableau de bord au démarrage du Raspberry Pi, à partir du répertoire MyChronoGPS, lancez la commande suivante :
pi@raspberrypi:~/MyChronoGPS $ sh ./CreateAutostart.sh

Utilisation de Google Maps API

Par défaut, MyChronoGPS utilise Leaflet pour les outils analyse et design track.
Si vous possédez une clé pour l'API Google Maps, vous pouvez demander à MyChronoGPS d'utiliser GoogleMaps à la place de Leaflet.

Pour çà, à partir du répertoire MyChronoGPS, effectuez les actions suivantes :
pi@raspberrypi:~/MyChronoGPS $ sh MyChronoGPS_GoogleMaps.sh
A l'édition du fichier /var/www/html/js/key.js par nano, modifiez la ligne
var myKey = 'Your_API_Key';
en remplacant le texte Your_API_Key par votre clé API puis sauvez le fichier modifié.

Utilisation de Leaflet API

Il est également possible de revenir à l'utilisation de Leaflet à la place de GoogleMaps.

Pour çà, à partir du répertoire MyChronoGPS, effectuez les actions suivantes :
pi@raspberrypi:~/MyChronoGPS $ sh MyChronoGPS_Leaflet.sh

Installation de corrections

Pour installer les corrections lancez les commandes suivantes :
(cette commande permet d'être sur le dossier racine, quelque soit le dossier courant)
pi@raspberrypi:~/MyChronoGPS $ cd $HOME
(les lignes suivantes pour nettoyer un correctif précédent)
pi@raspberrypi:~ $ sudo rm -rf temp
pi@raspberrypi:~ $ mkdir temp
(installation du correctif)
pi@raspberrypi:~ $ cd temp
pi@raspberrypi:~/temp $ git clone https://github.com/jfk93-fr/MyChronoGPS.git
pi@raspberrypi:~/temp $ cd MyChronoGPS
pi@raspberrypi:~/temp/MyChronoGPS $ sh correctif
En cas de problème, lancez la procédure d'installation de MyChronoGPS
pi@raspberrypi:~/temp/MyChronoGPS $ cd $HOME/MyChronoGPS
pi@raspberrypi:~/MyChronoGPS $ sh MyChronoGPS.sh
